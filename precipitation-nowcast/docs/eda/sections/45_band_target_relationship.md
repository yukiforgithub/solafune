# 45. バンド × 降水量の関係（輝度温度 vs 降水）

> Phase 1（物理ベースライン）の目星付け、および Phase 2（多バンド GBDT）の特徴量選択のための関係分析。
> サンプルベース推定（衛星別に 300〜800行 × 各300画素を抽出、`cv2.INTER_AREA` で 41×41 へ縮約して画素対応）。値は ±0.03 程度の標本誤差を含む。
> 図: [band_target_spearman.png](../figures/band_target_spearman.png) / [window_tb_rr_calibration.png](../figures/window_tb_rr_calibration.png) / 数値: [band_relationship_stats.json](../figures/band_relationship_stats.json)
> 関連: [40_input_bands.md](./40_input_bands.md)（バンド値域・死にバンド）, [10_target_distribution.md](./10_target_distribution.md)（ゼロ過剰）, [docs/phase1/](../../phase1/)（窓バンドのfit本体）

## 要点

1. **放射系（IR/WV）は全バンド・全衛星で降水と負相関**（低DN=冷たい雲頂=強雨）。§40 で保留だった「DNの向き」を確定。
2. **可視/近赤外は単独ではほぼ無力**（|ρ|≲0.2）、しかも**夜間は相関消失**。Phase 1 を IR/WV で組むのは正しい。
3. シグナルは **IR/WV 群に広く分布**（相互高相関）。単一窓バンドでほぼ取り切れ、B11(雲相 8.6µm)・WV群が**わずかに独立情報** → Phase 2 多バンドの上乗せは「中程度」と見積もる。
4. **split-window 生差分は衛星依存**：GOES のみ有効（ρ≈−0.18）、himawari 弱・meteosat 無効。
5. **Tb→RR は単調減少の滑らかな曲線** → lookup / べき乗則 / 指数則のいずれもフィット可（Phase 1 で比較中）。

## 1. 全16バンドの降水との相関（Spearman ρ）

降水画素を含む全画素での順位相関。負＝「DNが低い（冷たい/暗い）ほど降水が多い」。

| 順位 | himawari | ρ | goes | ρ | meteosat | ρ |
|---|---|---|---|---|---|---|
| 最強(負) | B11(8.6µm 雲相) | −0.55 | C14(11.2µm窓) | −0.49 | ir_87(8.7µm) | −0.40 |
| | B13/B14(10.4/11.2窓) | −0.55 | C13(10.3µm窓) | −0.49 | ir_105(10.5窓) | −0.40 |
| | B15(12.4) | −0.55 | C15(12.3) | −0.49 | ir_123(12.3) | −0.40 |
| | B07(3.9) | −0.53 | C11(8.4) | −0.49 | wv_73(7.3) | −0.39 |
| | B08–B10(WV 6–7µm) | −0.49〜−0.53 | C07–C10 | −0.42〜−0.48 | ir_38/ir_97/wv_63 | −0.33〜−0.39 |
| 反射系 | B01–B06(VIS/NIR) | +0.06〜+0.12 | C01–C06 | +0.10〜+0.20 | vis/nir | ≈0 |

- **放射系の帯（B07–B16 / C07–C16 / ir・wv群）が一様に強い負相関**。相互に高相関なので、どれか1本（IR窓）でほぼ代表できる。
- **可視/近赤外は弱い正相関**（明るい雲＝多少の雨）だが単独では実用域に届かない。GOES の C04(1.37µm 巻雲) が +0.20 とやや目立つ。
- 相関の強さ：**himawari > goes > meteosat**。meteosat は乾燥でレンジが狭く弱い（[20_stratification.md](./20_stratification.md) と整合）。

## 2. 昼夜別（可視明度の中央値で擬似分割）

各画像の可視バンド平均DNで day/night を代理分割した時の相関。

| 衛星 | IR窓 ρ（昼） | IR窓 ρ（夜） | 可視B1 ρ（昼） | 可視B1 ρ（夜） |
|---|---|---|---|---|
| himawari | −0.502 | −0.604 | +0.245 | **+0.040** |
| goes | −0.557 | −0.515 | +0.387 | **+0.018** |
| meteosat | −0.406 | −0.466 | +0.193 | **+0.019** |

- **可視は夜間に相関がほぼ消える**（昼の +0.2〜+0.4 → 夜 +0.02〜+0.04）。可視を使うなら**昼夜フラグ（または太陽天頂角）必須**。
- **IR窓は昼夜とも安定**（むしろ夜にやや強い）。**24時間使えるのは IR/WV**。
- → モデルには昼夜情報を明示し、夜間は IR/WV のみで成立する設計にする（[40_input_bands.md](./40_input_bands.md) の決定と一致）。

## 3. IR窓 Tb→RR キャリブレーション（Phase 1 の核）

窓バンドDN（低い=冷たい雲頂）に対する条件付き平均降水 E[y|DN] は **単調減少の滑らかな曲線**（図 [window_tb_rr_calibration.png](../figures/window_tb_rr_calibration.png)）。himawari の例（8DN幅ビン）:

| 窓DN帯 | 平均RR(mm/hr) | ゼロ率 |
|---|---|---|
| 0–16（最も冷たい） | ~5.5 | 8.8% |
| 32–48 | ~1.2 | 36% |
| 96–112 | ~0.35 | 64% |
| 160–176 | ~0.11 | 86% |
| 224–256（暖かい地表） | ~0.01 | 98% |

- 形状は **指数/べき乗で良く近似**でき、Auto-Estimator 型（RR = a·exp(−b·DN) など）が素直。Phase 1 で lookup / べき乗則 / 指数則を地域CVで比較中。
- 強雨は最冷帯（低DN）に集中 → **裾の較正が RMSE 改善の主戦場**（[10_target_distribution.md](./10_target_distribution.md) と整合）。
- 注意：物理単位（K）への逆変換ゲイン/オフセットは GeoTIFF に無いため、**衛星内の相対DN**でフィットし衛星間オフセット差は許容（[40_input_bands.md](./40_input_bands.md) line 95）。

## 4. split-window 差分（BT10.4 − BT12.3 相当）

生DN差分と降水の Spearman：**goes −0.18 / himawari +0.11 / meteosat −0.00**。
- 単独信号としては **GOES でのみ有効**。LH24 ディスカッションの「split-window 最良」は3バンド組合せ＋U-Net の空間学習込みの結果であり、**生差分単独**の寄与とは別物。
- → Phase 2 では split-window 差分を**特徴量の1つ**として入れるが（特にGOES）、過度な期待はしない。氷水経路の物理的意味は残るので CNN の入力チャネルとしては保持価値あり。

## Phase 1 / Phase 2 への含意

- **Phase 1**：IR窓単バンドで「単調な Tb→RR」を当てるのは妥当。全体平均 honest 1.4048 をどれだけ下回るかが基準値。単バンドの上限は lookup（≒キャリブレーション曲線そのもの）。
- **Phase 2（GBDT）**：可視/近赤外は昼のみ弱信号 → **IR/WV を主特徴**に、可視は昼夜フラグと併用。多バンドの上乗せは中程度（IR/WV群が相互高相関のため）。B11(雲相)・WV群・split-window(GOES)・3.9µm が差分情報の候補。空間統計（近傍 min/std/Z）と昼夜・天頂角で底上げする。
