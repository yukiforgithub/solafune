# EDA 10 — ターゲット降水分布の特性化

> Phase 0（EDA & パイプライン疎通）。ターゲット **GPM-IMERG 降水量（mm/hr）** の分布特性を確定し、損失設計・後処理・較正への含意を整理する。
> 集計対象は TRAIN の全 **40,686 ファイル**（各 41×41=1,681 画素、合計 **68,393,166 画素**）。
> 全体集計は `eda_cache/target_stats.parquet`（ファイル単位の合計/カウント）から、画素レベルのヒストグラム・正確な裾分位は raw target tif を **2,000 枚ランダム抽出**（seed=42、3,362,000 画素）して算出。
> 数値の生データ: [`eda_cache/target_dist_summary.json`](../../../eda_cache/target_dist_summary.json)

---

## 1. 要約（3行）

1. **極端なゼロ過剰**。正確に 0 の画素が **82.07%**、<0.1mm まで含めると **85.1%**。降水は「ほとんど起きていない」のが支配的な状態。
2. **正の降水はロングテール**。有雨（>0.1mm）の歪度は raw で **5.35**、平均 1.80 / 標準偏差 3.06 mm/hr、画素最大 **96.5 mm/hr**。`log1p` で歪度 5.35→1.28・尖度 47.4→1.43 とほぼ対称化される。
3. **「全 0」でも RMSE=1.4324**、「全平均(0.2886)」で 1.4030。原点はゼロ過剰では下がりきらない（後述）。RMSE を本気で下げるには **無降水を 0 と当てる × 強雨の裾を大外ししない** の両立が必須。

## 2. ビン別割合（全画素・全 40,686 ファイル）

![ビン別割合](../../../eda_cache/fig_target_bins.png)

| ビン (mm/hr) | 画素割合 | 累積（以上側）| 備考 |
|---|---:|---:|---|
| `==0`（正確なゼロ） | **82.07%** | 100% | 無降水が支配的 |
| `(0, 0.1)` | 3.08% | 17.93% | ごく弱い降水/ノイズ域 |
| `[0.1, 1)` | 8.16% | 14.85% | 有雨（>=0.1）はここから |
| `[1, 5)` | 5.32% | 6.69% | 中程度 |
| `[5, 10)` | 0.99% | 1.37% | 強雨 |
| `[10, 20)` | 0.31% | 0.38% | 非常に強い雨 |
| `>=20` | **0.079%** | 0.079% | 極端事象（裾） |

- **無降水（==0）= 82.07%**、**<0.1mm = 85.1%**。「有雨」を **>=0.1mm** と定義すると **14.85%** のみ。
- 5mm/hr 以上（強雨）は全画素の **1.37%**、20mm/hr 以上は **0.079%**（5.4万画素）にすぎない。にもかかわらず二乗誤差ではこの裾が RMSE を支配する（§5）。

> 注: cache の `eq0/lt01/ge01/ge1/ge5/ge10/ge20` は各ファイル 1,681 画素にわたる **カウントの合計**。ビン割合は全ファイル合計後に総画素数で割って算出（サンプルではなく全数）。

## 3. 正の降水の分布と log1p 対称化

![log1p ヒストグラム](../../../eda_cache/fig_target_log1p_hist.png)

- 左（生スケール・正の降水のみ・y 軸対数）: 0 近傍に集中し、指数的に減衰する典型的なロングテール。
- 右（log1p スケール）: 単峰でほぼ対称な釣鐘型に変換される。

**有雨（>0.1mm）条件付き分布**（サンプル 2,000 枚）:

| 指標 | 生スケール | log1p 変換後 |
|---|---:|---:|
| 平均 | 1.80 mm/hr | — |
| 標準偏差 | 3.06 mm/hr | — |
| **歪度** | **5.35** | **1.28** |
| **尖度** | **47.4** | **1.43** |

- `log1p` で歪度が 5.35→1.28、尖度が 47.4→1.43 と劇的に縮小 → **回帰ターゲットは log1p(降水量) を採るのが妥当**。MSE 損失は正規性を前提に効くため、log 空間で学習する方が中〜強雨の勾配が安定する。
- 全画素（ゼロ込み）では log1p でも歪度 12.7→4.38 と非対称が残る。これは log 変換ではゼロ過剰を解消できないためで、**「降るか/降らないか」と「降る量」を分けて扱う 2 部構成**（分類器 × 有雨回帰）が log 変換と相性が良いことを示す。

## 4. 裾（強雨側）

| 分位 | 全画素 | 有雨（>0.1）条件付き |
|---|---:|---:|
| p50 | 0.00 | 0.74 |
| p90 | 0.32 | 4.40 |
| p95 | 1.28 | 6.67 |
| p99 | 5.40 | 14.96 |
| p99.9 | 17.11 | 32.46 |
| p99.99 | 34.55 | 50.31 |
| **max** | 77.6（サンプル）／**96.5（全数）** | 77.6（サンプル） |

- 全画素 p99 = 5.4mm/hr、p99.9 = 17.1mm/hr。**上位 0.1% の画素が二桁 mm/hr** を占める急峻な裾。
- 全数での画素最大は **96.5 mm/hr**（cache `vmax` の最大）。サンプル 2,000 枚では 77.6mm/hr まで観測。
- 強雨は希少だが二乗誤差で支配的（§5）。**裾の過小評価を較正（isotonic 等）で補正する余地が最大の改善源**。

## 7. RMSE への含意

### 7-1. なぜ「全 0」でも RMSE=1.4324 か
RMSE は二乗平均平方根なので、全予測を 0 にしたときの誤差は **降水量の二乗平均の平方根 = √(Σv²/n)** に等しい。

- ゼロ過剰（82%）ゆえ大半の画素は誤差 0 だが、残り 18% の有雨画素、とりわけ希少な強雨（p99.9=17mm, max=96.5mm）の **二乗** が効いて 1.4324 まで押し上げられる。
- 全予測=平均(0.2886) でも 1.4030。ゼロ過剰下では「定数で当てる」戦略の改善幅はわずか **2%（1.4324→1.4030）**。**この 2 つが以後の改善を測る原点**。

> 直感: RMSE 1.43 の正体は「82% を完璧に当てても、残り 18% の、特に上位 0.1% の強雨を外すと二乗で大きく罰される」構造。ゼロ過剰は RMSE を **下げてくれない**（むしろ強雨の希少性が裾の学習を難しくする）。

### 7-2. ゼロ過剰への対処
- **2 部構成（分類器 P(rain) × 有雨回帰）** または **Tweedie 損失** が素直。82% のゼロをまず正しく 0 にするだけで、誤検出由来の二乗誤差を抑えられる。
- 予測後の **微小値のゼロ丸め**（しきい値は CV 最適化）と **負値クリップ→0** は必須の後処理。

### 7-3. log 変換と元スケール較正
- 有雨は log1p でほぼ対称化（歪度 5.35→1.28）するため、**log1p 空間で回帰**すると学習が安定する。
- ただし **評価は元スケール（mm/hr）の RMSE**。log 空間で MSE 最小 ≠ 元スケールで RMSE 最小（Jensen の不等式により expm1 で戻すと系統的に過小評価が出やすい）。
  - 対策: 予測を expm1 で戻したあと、**isotonic 回帰等で元スケール較正**。特に **強雨側の過小評価補正**が RMSE への寄与が最大（裾が二乗で効くため）。
  - しきい値（ゼロ丸め）・較正パラメータはすべて **手元 CV で最適化**（Public 35% への過適合を避ける）。
